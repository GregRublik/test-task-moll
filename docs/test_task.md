# Тестовое задание: Python-разработчик

## Контекст

Компания работает с данными о банкротстве физических лиц из реестра ЕФРСБ (bankrot.fedresurs.ru). Доступ к данным предоставляется через платный API — каждый успешный запрос тарифицируется, поэтому экономия запросов критически важна.

Нужно написать скрипт, который по списку ФИО с датами рождения выгружает данные из API и сохраняет их в CSV-файлы.

---

## Задача

Написать Python-скрипт `main.py` для выгрузки данных физических лиц через HTTP API.

### Входные данные

Текстовый файл `input.txt` — список физлиц, по два поля на каждого:

```
Иванов Иван Иванович
15.03.1985

Петрова Мария Сергеевна
22.07.1990
```

### Выходные данные

Папка `output/` с CSV-файлами. API возвращает три связанные сущности — физлицо, сообщения о банкротстве и лоты торгов. Структуру файлов и набор полей определи самостоятельно.

---

## API

Базовый URL: `https://parser-api.com/parser/fedresurs_api/`  
Авторизация: параметр `key=` в каждом запросе (передаётся в `.env`)

Полная документация эндпоинтов — в файле `api_docs.txt`.

---

## Обязательные требования

**Инкрементальный режим** — ключевое требование:
- Запросы к API платные, каждый `success=1` тарифицируется
- При повторном запуске с тем же `input.txt` скрипт не должен тратить лишние запросы на данные, которые уже были получены

---

## Что оценивается

- Корректность инкрементальной логики — главный критерий
- Читаемость и структура кода
- Обработка граничных случаев (человек не найден, несовпадение ДР, пустые страницы)
- Ясность итогового вывода в stdout

## Что не оценивается

- Тесты
- Typing / аннотации типов
- Асинхронность
